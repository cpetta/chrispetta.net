Website Development

Kuzmaclass - Homepage

The kuzmaclass website provides useful information for our students and their parents. I was tasked with prototyping a version of the site that reduced the complexity of the code, added a student showcase, and made the site touchscreen friendly. I reduced the complexity of the code by adding comments and removing unused or redundant code. I created the showcase section using flexbox’s in HTML and CSS. Finally, I made the site touchscreen friendly by increasing the size of hyperlinks, creating a mobile friendly navigation menu, and modifying the layout to work at mobile resolutions.

Kuzmaclass - Sandbox

The sandbox is a webpage that was created to act as a portal to student projects. It started off as a simple page, with hyperlinks to student webpages. After a while, it became hard to find specific projects, and we needed a way to sort what was there. When redesigning the sandbox page, our primary goal was to allow all projects to be searched and sorted. Some secondary goals included storing the information for each student project in a way that would make them more portable so that we can easily switch to other technologies in the future. And finaly to organize the student projects into logical groups for easier sorting. These goals were accomplished by storing the student project information as an array of JSON objects. The display of each project ‘card’ is handled using JavaScript, with a focus on the code being highly reusable.

Sandpaper

Sandpaper is a tool I developed to make building and optimizing websites easier and faster. It was heavily inspired by a code quality checking tool called XO. Sandpaper automates the process of checking code quality, which can help developers avoid common errors in HTML, CSS, and JavaScript. It optimizes assets to improve page load times by up to 30%. Additionally, it automatically substitutes newer code that is not supported in older browsers with code that is supported by older browsers.
