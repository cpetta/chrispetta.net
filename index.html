Website Development

Kuzmaclass - Homepage

The kuzmaclass website provides useful information for students and parents, displays. I was asked to clean up some aspects of the site. Specifically, to reduce the complexity of the code, add a student showcase, and make the site touchscreen friendly. I reduced the complexity of the code by adding comments, removing anything that was unused, or redundant. The showcase was created using standard HTML5 and CSS3. And the site was made touchscreen friendly by increasing the size of hyperlinks, creating a mobile friendly navigation menu, and modifying the layout to work at mobile resolutions.

Kuzmaclass - Sandbox

The sandbox is a webpage that was created to act as a portal to student projects. It started off very plain, without anything but links to other webpages. After a while, it became hard to find specific projects, and we needed a way to sort what was there. The current sandbox page allows all projects to be searched and sorted. The primary goals of this project were to store the information for each student project in a way that would make them more portable. Allow them the list of student projects to be sorted. And organize the student projects into logical groups. These goals were accomplished by storing the student project information as an array of JSON objects. The display of each project ‘card’ is handled using JavaScript, with a focus on the code being highly reusable.

Sandpaper

Sandpaper is a tool I developed to make building and optimizing websites easier and faster. It was heavily inspired by a code quality checking tool called XO. Sandpaper automates the process of checking code quality, which can help developers avoid common errors in HTML, CSS, and JavaScript. It optimizes assets to improve page load times by up to 30%. Additionally, it automatically substitutes newer code that is not supported in older browsers with code that is supported by older browsers.
